  • MMI Slideshow - Goals


    Why?

    • Half the Internet is slideshows. The other half is web apps with images.
    • HTML5 picture element really needs the help of front and/or back end scripting.
    • Since pictures are half the Internet (and also the other half), we can optimize a lot by using the picture element.

    Front End

    • Configurable, lazy loading of all images content with an data-src attribute.
    • Optional thumbnails from a sprite.
    • Usual JavaScript slideshow stuff:
      • Pagination, captions, credits, transitions, responsive, swipes and keyboard input, page numbering, play/pause button, show and hide navigation and captions ... and tests!

    Back End

    • Create slideshow automatically from a directory of images.
    • Create slideshow using the picture element automatically from a directory of images.
    • Load photos and photo meta-data from a photos.json file.
    • Create thumbnail sprite.png automatically.

  • MMI Slideshow - Design options


    Small, like Bootstrap Carousel

    • Good: Nice separation of concerns.
    • Bad: Why do I have to put the navigation elements into the HTML!?

    Big, like LA Times Slideshows

    • Good: Works beautifully. Sends different images based on screen resolution.
    • Bad: Wow, that's some complicated HTML.

  • MMI Slideshow - Challenges


    • CSS:
      • Trying to figure out how browsers (are supposed to) calculate an element's height. There's 10 cases: W3C Height Spec
      • Transitions for modern browsers. Javascript for older ones.
      • Use class names. Never ids (for a widget).
      • Child selector is safer than decendent selector.
      • Pseudo namespaces
    • JS: Pretty easy in comparison.
      • Trying to use reactive techniques to reflect state rather than storing it.

  • OCR Panel and ClusterManager


    Goals

    • Easy for reporters to create.
    • Lazy loading: The panel loads after the rest of the page.
    • Centralized: A loader file that's never cached points to the newest files that may be cached.
    • Resizeable since maps are often sidebar elements of a story.

  • OCR Panel Design Options


    Static, single page

    • Good: Tried and true method for Google maps
    • Bad: All that work for only one map!?

    A widget

    • Good: Reporters will help me make thousands of maps!
    • Bad: No one (that I could find) had done this.
